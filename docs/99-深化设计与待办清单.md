# 99 - 深化设计与待办清单

为了能让《Dungeon Tavern》的 MVP 版本顺利进入程序开发与数据配置阶段，以下核心系统在逻辑和数值上已经全部完成深化与闭合。

---

## 📌 已深化的四大核心板块状态总结

### 1. 【酿酒与材料】MVP 自定义组合与配方数值表 (核心循环的骨架) - [已完成]
*   **确认原则**：
    *   **完全自定义组合**：无怪味酸败液惩罚。酿酒机制由玩家完全自定义投入的材料和比例。
    *   **材料口味倾向**：已在 [09-口味系统.md](09-%E5%8F%A3%E5%91%B3%E7%B3%BB%E7%BB%9F.md) 中完整定义并定位了 16 种核心口味倾向。
    *   **口味融合加权与偏好**：酒水口味根据原材料口味加和。各智慧种族的喜恶口味偏好加权矩阵已确立。
    *   **试口与不喝酒设定**：已在 [03-酿酒与材料系统.md](03-%E9%85%BF%E9%85%92%E6%9D%90%E6%96%99%E7%B3%BB%E7%BB%9F.md) 补充“酿酒前后强制试口，平时滴酒不沾”的世界观逻辑，以合理解释酿造一次解锁材料口味的图鉴机制。
    *   **材料掉落与风味解耦**：已在 [10-酿酒材料大表.md](10-%E9%85%BF%E9%85%92%E6%9D%90%E6%96%99%E5%A4%A7%E8%A1%A8.md) 中将魔物掉落物风味重新分配。所有种族爱喝的风味绝对不会出现在它们自己掉落的材料上。
    *   **400回合探索时限与连轴搜刮 (Overnight Looting)**：白天探索时限为 400 行动回合。准时回城可正常进行夜晚营业；超时未归视为留在野外连轴搜刮，当晚营业停摆（无收益），且深夜怪物活性暴动（命中率 $+10\%$，伤害 $+20\%$，暴击 $+5\%$）（详见 [04-探索与地图生成.md](04-%E6%8E%A2%E7%B4%A2%E4%B8%8E%E5%9C%B0%E5%9B%BE%E7%94%9F%E6%88%90.md)）。
    *   **10 种经典配方配比大表**：已在 [13-经典配方与酒谱大表.md](13-%E7%BB%8F%E5%85%B8%E9%85%8D%E6%96%B9%E4%B8%8E%E9%85%92%E8%B0%B1%E5%A4%A7%E8%A1%A8.md) 中配置好 10 种经典酒谱，去除了黄金特权加成。
*   **当前缺失**：无（Demo暂不开发，保留至后续版本）。

### 2. 【怪物社交】NPC 记忆标签与关系逻辑流 (核心卖点) - [已保留至后续版本]
*   **确认原则**：
    *   **好感双轨机制**：人类顾客无好感度，纯商业买卖；怪物顾客只要来喝酒必定增加好感度，不论最终赠予多少亮铁片。
    *   **个体向群体传递**：个体好感变动向下取整 10% 累积到对应的整体势力声望中。
    *   **隐性好感表达**：界面 UI 隐藏任何好感或声望的数值进度。顾客对酒的态度和对老板的好感完全由四档语气反差强烈的台词与神态动态体现。
    *   **声望阶梯与收益矩阵**：划分警惕、中立、友好、信任、崇敬、生死之交 6 大等级，增加来店率、钱包乘数及赠礼概率（参见 [12-满意度与价格结算系统.md](12-%E6%BB%A1%E6%84%8F%E5%BA%A6%E4%B8%8E%E4%BB%B7%E6%A0%BC%E7%BB%93%E7%AE%97%E7%B3%BB%E7%BB%9F.md)）。
    *   **声望赠礼大表 (Gift Table)**：喝爽且判定成功时，各种族顾客会在离开酒馆时顺手赠予本族专属的酿酒材料。
*   **当前缺失**：无。

### 3. 【战斗系统】百分比命中闪避与投降机制移除 - [已完成]
*   **确认原则**：
    *   **完全移除投降与谈判**：地底战斗仅分生死，彻底移除了 Morale（意志）投降机制以及求饶、谈判等玩法。
    *   **百分比命中与闪避对抗**：彻底弃用 D20 骰子判定。改为百分比命中与闪避的直接减算。
        $$\text{最终命中率 } P_{hit}(\%) = 75\% + \text{攻方命中率加成}(\%) - \text{防方闪避率}(\%)$$
        最终命中率 $P_{hit}$ 边界锁死为 $5\% - 95\%$。
    *   **负重行动 Tick 衰减与装甲熟练度特权**：在格子回合制中，负重 $70\%-100\%$（重荷）每次移动一格被动消耗 $2$ 个行动 Tick；负重超过 $100\%$（超重）则移动力强行归 0（详见 [05-战斗系统.md](05-%E6%88%98%E6%96%97%E7%B3%BB%E7%BB%9F.md)）。轻甲与重甲熟练度均采用挨打（承受最终伤害被击中）双轨获取机制。轻装与重装在一阶、二阶、三阶均解锁不同的专属被动额外特权加成（如一至三阶轻甲提供轻盈步法、侧身闪避、疾风闪跃特权；重甲提供装甲加固、冲击缓冲、不屈壁垒特权）（详见 [05-战斗系统.md](05-%E6%88%98%E6%96%97%E7%B3%BB%E7%BB%9F.md)）。
*   **当前缺失**：无。

### 4. 【装备系统】双轨互斥携带与战力自然隔离 - [已完成]
*   **确认原则**：
    *   **去出身特质**：完全删除了玩家的老板初始出身特质背景。玩家在开局时属于白板，实力完全由后天发育决定。
    *   **战力自然隔离**：移除了任何物理环境抗性与生态锁设定。火山、墓地等高阶地图直接开放，利用怪物的超高血量与基础攻击力差（血量在 $1000-2200$，伤害在 $120-320$）对低端玩家形成自然的战力阻隔。
    *   **资源双轨互斥携带**：顾客生成时要么只携带随机钱包金币（50%概率），要么金币为 0 且携带一件随身装备（50%概率）。
    *   **盲盒赠予**：当未携带亮铁片的怪物顾客喝爽（$\Delta \ge 4$）时，因无铁片，会直接将随身携带的装备赠予老板作为谢礼。赠予的装备完全从全局库纯随机抽取（盲盒逻辑），与势力声望高低无任何概率挂钩。
*   **当前缺失**：无。

### 5. 【技能系统】双轨领悟与装备媒介 - [已完成]
*   **确认原则**：
    *   **双轨领悟获取**：技能获取完全依靠武器熟练度级别（Lv3/Lv8/Lv15）和主属性数值门槛（15/35/70点）在夜间回到酒馆时被动自动解锁，无传统加点树。
    *   **搭载媒介限制**：技能领悟后必须手持对应类型的武器作为释放硬件媒介，方可在战斗中激活使用，否则置灰禁用。
    *   **33个奇幻风格技能大表**：已在 [15-技能与领悟系统.md](15-%E6%8A%80%E8%83%BD%E4%B8%8E%E9%A2%86%E6%82%9F%E7%B3%BB%E7%BB%9F.md) 中配置了 10 种武器及 1 种徒手流派、一至三阶共 33 个经典奇幻风格的战斗技能。
*   **当前缺失**：无。

### 6.  **【代码与技术架构 (Godot 4.x + GDScript)】模块解耦与场景树设计** - [已完成]
*   **确认原则**：
    *   **架构规范化**：已在 [16-技术架构与代码设计.md](16-%E6%8A%80%E6%9C%AF%E6%9E%B6%E6%9E%84%E4%B8%8E%E4%BB%A3%E7%A0%81%E8%AE%BE%E8%AE%A1.md) 中完整确立了基于 Godot 4.x Scene/Autoload 与 GDScript 的层级结构。
    *   **Node-Component组合**：明确战斗与网格移动采用组件化 Node 挂载及 `ICombatant` / `IDamageable` 接口限定的强类型风格。
    *   **决策结算实现**：定义了 `PricingEvaluator` 与 `PlateExchanger` 作为独立结算类，彻底闭合人类标价评估与怪物亮铁片折换的逻辑决策树。
    *   **数据模型与扁平存档**：定义了 `BrewingKeg` 发酵 Keg 结构，并设计了完全支持 JSON 序列化/反序列化的扁平 GDScript 运行时存档架构。
*   **当前缺失**：无。
