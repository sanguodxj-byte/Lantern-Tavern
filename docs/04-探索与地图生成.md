# 04 - 探索与地图生成

> **文档换算说明（重要）**：本文档原始设计基于格子回合制；当前项目已确认转向 **3D 实时 ARPG**（参考《Barony》式体验）。文中所有"回合 / Tick / 格"均已按以下基准折算为实时表述，具体数值为**设计建议，需按手感调参**：
> - **时间**：1 行动回合 / Tick ≈ 0.5 秒。故"消耗 N 回合动作"→前摇 N×0.5s；"冷却 N 回合"→冷却 N×0.5s；"持续/内 N 回合"→状态持续 N×0.5s。
> - **距离**：技能与位移 **1 格 ≈ 2 米**（地图生成网格单元 = 3.0 米，见 16-技术架构 §6.4）。故"击退 N 格"→击退 N×2 米；"射程/范围 N 格"→N×2 米。
> - 已永久**移除**的格子回合制规则见 [99-深化设计与待办清单.md](99-深化设计与待办清单.md) 的"已取代设计存档"一节（仅作旧设计参考，不直接接入主流程）。
>


## 1. 核心原则：搜打撤机制
在《Dungeon Tavern》中，探索的本质是**资源搜集与风险博弈**，而非传统的杀怪刷级。

### 1.1 资源博弈流程
1.  **准备与进入**：玩家清空背包，选择携带有限的消耗品（酒、干粮）进入地下城。
2.  **搜集与判断**：在行进中搜集材料、遭遇 NPC、触发随机事件。背包空间有限，玩家必须在“高价值酿酒材料”与“保命道具”之间进行取舍。
3.  **撤离判断**：随着探索深入，敌人的威胁和自身的负面状态会不断累积。玩家需要随时判断“是继续深入以获取传说级材料，还是立刻安全撤离”。
4.  **返回酒馆**：只有通过地图中的特定撤离点安全返回，才能将本次获得的材料与搜刮到的随机装备顺利存入酒馆仓库。

### 1.2 撤离失败与死亡惩罚机制 (Failure Penalty)
在地下城中，普通人类（如冒险者）被击杀会迎来真正的生命终结。然而作为玩家，倒下时得益于神秘契约与特权庇护，不会导致永久死亡（不删档）。当前项目采用 **Elona-like 长期成长 + 远征撤离循环**，失败惩罚应打击“本趟收益”，而不是摧毁玩家长期构筑：
*   **保留长期成长**：玩家的主属性、武器熟练度、已领悟技能、酒馆建设、NPC 关系、区域解锁、仓库既有资产都不会因为撤离失败而丢失。
*   **已装备随机损坏**：玩家当前已装备 / 已配置的防具、武器、防盾中，随机 1 件进入损坏或低耐久状态。损坏装备不会消失，可在回到酒馆后通过铁匠或材料修理。
*   **背包战利品全丢**：除已装备 / 已配置的防具与武器外，本次远征背包内尚未安全带回酒馆的材料、临时采集物、未入库战利品全部丢失。
*   **强制遣送酒馆**：玩家会在酒馆柜台后的躺椅上醒来，时间强制直接跳跃流转至当晚的酒馆经营期。
*   **酒馆生态损失**：由于白天收获不足或全无收获，酒馆会面临“库存短缺”。这可能导致当晚无法供应高品质酒水，造成营业额下滑。

### 1.3 撤离方式与交互判定 (Escape & Extraction)

为了维持“搜打撤”极高的博弈感和逃跑时的紧张气氛，玩家无法在野外直接传送，必须通过地牢中的物理交互点撤离：

1.  **地底铁索升降井 (Extraction Shafts)**：
    *   **随机生成位置 (Procedurally Spawned Positions)**：由于本作的地下城探索地图是由 WFC 算法随机生成的，因此每张地图中 **2-3 个升降井的刷新位置也是完全随机的**。玩家无法通过“背图”直接撤离，必须在探索中优先摸索定位它们，增加了因负重卡死的博弈风险。
    *   **逃离判定 (1.5 秒读条防秒退)**：玩家必须移动至升降梯网格内，按下交互键拉动拉杆。拉动后进入**“撤离等待状态”（正常状态下需要约 1.5 秒）**。
    *   **负重状态对撤离的影响**：
        *   **重荷状态（负重 $70\% - 100\%$）**：由于身手受到行李拖累，拉动拉杆后的“撤离等待状态”**延长为约 2 秒**，同时等待期间的闪避率额外降低 $60\%$。
        *   **超重状态（负重 $> 100\%$）**：由于无法位移（定身），玩家无法通过移动踩入升降梯网格。必须扔掉物品使负重降低至 $100\%$ 以下，或者使用昂贵的`[逃脱符石]`原地强行开启传送门。
    *   **限制条件**：在等待期间，玩家不能移动，闪避率大幅削减。周围的怪物会被拉动闸门的铁链声吸引并群起攻之。玩家必须在确认周围安全后，才能拉闸撤离。
2.  **随机撤离点：古代缝隙 / 逃脱石阵 (Random Escapes)**：
    *   **刷新规则**：在深层的高难地图（如墓园、沼泽），地图深处会在探索进行到中后期时，在随机位置冒出带光晕的“古代时空缝隙”（或地底逃生石阵）。
    *   **单人限定**：每个随机缝隙是一次性的，只允许玩家使用一次。玩家如果处于“超重状态”，同样无法通过走位踩入缝隙网格进行交互。
3.  **道具撤离：逃脱符石 (Escape Runes)**：
    *   玩家击杀地图深处的精英怪物或开金宝箱时，极低概率掉落一次性消耗性道具 `[逃脱符石]`。使用后可以在原地直接开启一个逃脱传送门。
    *   **负重对符石读条的影响**：
        *   正常或重荷状态下，使用符石需要原地站立读条 **2.5 秒**。
        *   超重状态下，使用符石的能量阻力极大，读条时间**延长为 4 秒**。

### 1.4 探索时间窗口与“野外连轴搜刮”博弈 (Overnight Looting)

为了让玩家体验极具张力的野外生存与经营收益的策略平衡，白天的地下城引入以下时限流转规则：

1.  **日探索时间片 (约 200 秒（≈3.3 分钟）实时探索软上限)**：
    *   玩家进入关卡后，该白天最多可在约 200 秒（≈3.3 分钟）的实时探索软上限内行动；超出后时间自动流转入夜（详见下方"野外连轴搜刮"机制）。
2.  **准时撤离与当天营业**：
    *   如果玩家在 约 200 秒（≈3.3 分钟）内成功通过升降井或缝隙撤离，即可在日落前返回酒馆，**正常进行当晚的顾客经营与收益结算**。
3.  **野外连轴搜刮 (Overnight Looting) 机制**：
    *   如果 约 200 秒（≈3.3 分钟）走完玩家仍未归，或者玩家主动决定留在野外不返回，**时间将在后台自动流转入夜**。
    *   **营业停摆惩罚**：酒馆当晚由于无人主持，**无法进行任何营业，当天酒馆收益强行归零**。不过由于员工对酒馆绝对忠诚且无偿服务，玩家不需要在当晚承担任何员工工资支出。
    *   **深夜怪物活性化暴动**：玩家不会被强制遣送，允许留在地下城深处连轴搜刮。但由于夜幕降临，**地下城所有怪物进入活性暴动状态：其命中率额外 $+10\%$，暴击率额外 $+5\%$，且基础物理/法术伤害加成 $+20\%$**，极易在野外暴毙。
    *   **跳过经营夜**：待玩家在野外继续呆满第二个白天并最终撤离后，时间将跳过中间错失的经营夜，直接计入下一次夜晚营业。

## 2. 怪物战力阶梯隔离机制 (Ecology-Strength Filter)

## 3. 地下城程序生成机制
为了保证探索的重复游玩价值（Replayability），地下城地图采用**结构化模板**与 **WFC (Wave Function Collapse / 波函数崩塌算法)** 混合生成。

### 3.1 生成模板分配

| 模块类型 | 生成方式 | 包含内容 |
| :--- | :--- | :--- |
| **固定框架 (Fixed Anchor)**| 手工设计与配置 | 区域领主的王座、剧情主线关键遗迹、特定剧情 NPC 的固定营地。 |
| **随机布局 (WFC Randomized)**| 波函数崩塌算法拼接 | 房间与通道的几何拓扑拼接，确保每次进入的路径布局完全不同，并生成合理的逻辑闭环（不会出现死路）。 |
| **动态分布 (Dynamic Spawning)**| 生态概率分布 | 酿酒植物的生长位置、野生魔物的巡逻路线、随机冒险者的篝火营地、隐藏宝箱与陷阱的位置。 |

### 3.2 WFC 生成规则
*   **地块拼接法则**：游戏将地图分解为不同的“地块单元”（如：走廊、十字路口、三岔路口、大小房间、死胡同）。WFC 算法会根据相邻地块的边缘拼接规则，在运行时动态崩塌出兼具美感与合理性的探索地图。
*   **钥匙与铁门强制映射约束 (Key-Gate Constraint)**：
    在 WFC 崩塌拼图的随机房间生成中，算法在执行崩塌前会优先读取玩家当前的库存状态。如果玩家背包或仓库中存有未使用的“某区域古代保险箱钥匙”，WFC 算法将强制把“需要该钥匙开启的古代锁闭铁门/密封宝藏区”地块模块注入随机拼接的“固定框架 (Fixed Anchor)”清单中，确保本张随机生成的探索地图中有且仅有 1 个对应的铁门保险箱房间生成，杜绝钥匙变成废品的玩家负面挫败感。

### 3.3 地形伤害与物理位移战斗空间 (Hazard Combat Layout)

当前版本的地下城是 **3D ARPG 像素地牢探索**，地图生成必须把“地形伤害 + 物理位移”视为核心战斗玩法，而不是单纯装饰陷阱。玩家已经拥有踢击、冲撞、抓取投掷、武器击退等位移手段；场景中也已有尖刺、酸液等地形伤害机制。因此，WFC 与动态分布阶段需要主动生成可被玩家利用的战斗地形，让玩家能够把敌人踢向尖刺、推入酸液、撞上墙面或障碍物完成高风险击杀。

1.  **危险地形不是随机噪声**：
    *   尖刺、酸液、裂隙、坠落边缘等危险地形必须作为“战斗锚点 (Hazard Anchor)”参与房间生成。
    *   每个危险锚点周围需要保留可读的站位空间，让玩家能观察、绕行、诱敌和执行踢击。
    *   禁止在入口、撤离点、宝箱交互点正脚下生成不可预期的即死危险。
2.  **踢击路线与击退通道 (Kick Lanes)**：
    *   危险地形前方应生成若干条 2-5 米的直线或斜向“踢击路线”，保证敌人可以被踢击、冲撞、战锤击退等效果推入危险区。
    *   踢击路线不能被过多小障碍物打断，但可以在边缘放置柱子、墙角、栅栏等作为碰撞伤害或卡位选择。
    *   房间中至少保留一条玩家可绕到敌人侧面或背后的路线，避免危险地形只对玩家构成负担。
3.  **风险与奖励绑定**：
    *   高价值材料、精英怪、宝箱、保险箱房间附近可以提高危险地形密度，形成“要奖励就要利用地形或承受风险”的空间博弈。
    *   新手区危险地形应少量、显眼、低密度；中后期区域可以组合尖刺 + 狭窄桥面 + 推拉敌人 + 巡逻路线，形成更明确的环境击杀谜题。
4.  **敌人与地形协同**：
    *   低智野怪可以直接追击玩家，容易被引诱到尖刺路线。
    *   高阶或智慧敌人应尝试绕开明显危险地形，或把玩家反向逼向危险区。
    *   具有冲撞、击退、抓取能力的敌人可以与危险地形组合，但必须给予视觉预警和可闪避空间。
5.  **可读性与公平性**：
    *   尖刺、酸液等伤害体需要明确视觉轮廓、音效反馈和受击反馈。
    *   玩家应能在进入危险房间前看见至少一部分危险地形，而不是被拐角后的隐藏尖刺无提示秒杀。
    *   地图生成校验必须保证主路径可通过，不要求玩家必须承受地形伤害才能到达撤离点或关键剧情点。
