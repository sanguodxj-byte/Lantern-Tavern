# 预览 vs 运行时差异排查工作流

本文档定义"建模预览看起来正常，游戏内骨骼颠倒/朝向错误/材质异常"问题的系统排查流程。

## 0. 问题背景

游戏运行时与预览工具之间存在 **5 层关键差异**，任何一层都可能导致视觉不一致：

| 层次 | 预览工具 | 游戏运行时 | 差异影响 |
|------|----------|-----------|---------|
| 材质 | 原始 GLB 材质 | `VoxelLightingAdapter` 适配后（toon 着色） | 颜色/高光/金属感不同 |
| 动画 | 导出 GLB 轨道数据 | 同一份已验收轨道（运行时不得修补） | 如果不同，资产或场景配置必须拒收并定位根因 |
| 场景 | 裸 GLB 实例 | 含 PhysicalBoneSimulator3D + 装备 | 骨骼物理/武器挂载不同 |
| 朝向 | 原始根节点旋转 | 场景可能覆盖旋转 | 模型朝向不同 |
| 环境 | 诊断工具自带灯光 | 地牢/酒馆环境光 | 整体明暗/氛围不同 |

## 1. 排查流程总览

```
发现差异
  │
  ▼
步骤1: 运行诊断工具（3个脚本）
  │    ├── runtime_vs_preview_diag.gd   → 分层对比报告
  │    ├── skeleton_facing_diag.gd      → 骨骼朝向报告
  │    └── animation_track_diag.gd      → 动画轨道报告
  │
  ▼
步骤2: 对照差异表定位问题层
  │    材质异常? → 第3节
  │    动画异常? → 第4节
  │    骨骼颠倒? → 第5节
  │    朝向错误? → 第6节
  │    武器位置? → 第7节
  │
  ▼
步骤3: 运行针对性单元测试
  │    runtime_vs_preview_diag_test.gd
  │
  ▼
步骤4: 修复后回归验证
       ├── 重新运行诊断工具
       └── 重新运行单元测试
```

## 2. 步骤1：运行诊断工具

### 2.1 运行时 vs 预览分层对比

```bash
# 诊断所有 _rig.glb 和默认场景
"D:/123/Godot_v4.7-stable_mono_win64.exe" --headless --path "D:/123/Lantern Tavern" \
  --script res://tools/runtime_vs_preview_diag.gd

# 仅诊断指定 rig
"D:/123/Godot_v4.7-stable_mono_win64.exe" --headless --path "D:/123/Lantern Tavern" \
  --script res://tools/runtime_vs_preview_diag.gd -- --rig=voxel_goblin_32px_rig.glb

# 仅诊断指定场景
"D:/123/Godot_v4.7-stable_mono_win64.exe" --headless --path "D:/123/Lantern Tavern" \
  --script res://tools/runtime_vs_preview_diag.gd -- --scene=goblin
```

输出：`reports/runtime_vs_preview/runtime_vs_preview_report.txt`

报告内容：
- **阶段1**：原始 GLB 节点树、骨骼信息、根节点旋转、材质快照
- **阶段2**：应用 VoxelLightingAdapter 后的材质快照（⚠️ headless 下会跳过）
- **阶段3**：导出轨道类型、关键帧数据与预览/运行时一致性对比
- **阶段4**：完整场景实例化的节点结构、BoneAttachment3D、PhysicalBoneSimulator3D

### 2.2 骨骼朝向诊断

```bash
"D:/123/Godot_v4.7-stable_mono_win64.exe" --headless --path "D:/123/Lantern Tavern" \
  --script res://tools/skeleton_facing_diag.gd

# 单个 rig
"D:/123/Godot_v4.7-stable_mono_win64.exe" --headless --path "D:/123/Lantern Tavern" \
  --script res://tools/skeleton_facing_diag.gd -- --rig=voxel_goblin_32px_rig.glb
```

输出：`reports/skeleton_facing/skeleton_facing_report.txt`

检查项：
1. 根节点朝向修正（180°/-90° Y 旋转）是否存在
2. 每根骨骼 rest pose 局部 Y 轴是否朝上（垂直骨骼）
3. 骨骼链连续性（parent tail 与 child head 是否重合）
4. 关键骨骼全局位置（Head 在上、Foot 在下、Hand.R 在右）
5. 网格 AABB 是否在地面以上

### 2.3 动画轨道诊断

```bash
"D:/123/Godot_v4.7-stable_mono_win64.exe" --headless --path "D:/123/Lantern Tavern" \
  --script res://tools/animation_track_diag.gd

# 单个 rig
"D:/123/Godot_v4.7-stable_mono_win64.exe" --headless --path "D:/123/Lantern Tavern" \
  --script res://tools/animation_track_diag.gd -- --rig=voxel_goblin_32px_rig.glb
```

输出：`reports/animation_track/animation_track_report.txt`

检查项：
1. 所有必需动画是否存在
2. 腿部骨骼（UpperLeg/LowerLeg）轨道类型是否为 ROTATION_3D
3. 关键帧数据是否有效（非 NaN、Quaternion 归一化）
4. 导出数据与运行时轨道类型/关键帧的一致性
5. kick/run/slash 关键动画的详细轨道检查

## 3. 材质异常排查

### 症状
- 游戏内模型全白/全黑
- 颜色与预览不一致
- 金属感消失或过强

### 排查步骤

1. **检查 VoxelLightingAdapter 是否应用**：
   - 报告阶段2 会显示"VoxelLighting后"材质快照
   - headless 模式会跳过适配，需在非 headless 下验证

2. **检查 vertex_color_use_as_albedo**：
   - `VoxelLightingAdapter.adapt_standard_material()` 保留原始设置
   - 如果 GLB 材质未开启此项，顶色不会作为 albedo

3. **检查 base_material 覆盖**：
   - `enemy.gd` 的 `base_material` 属性已废弃
   - 如果场景仍配置了 `base_material`，检查是否产生干扰

4. **修复方法**：
   - 在 Blender 导出时确保材质设置正确
   - 不要手动覆盖 `material_override`，让 VoxelLightingAdapter 统一处理

## 4. 动画异常排查

### 症状
- kick/run 动画腿部不动
- 动画播放时骨骼旋转无效果
- 预览中动画正常但游戏内异常

### 排查步骤

1. **检查轨道类型**：
   - 运行 `animation_track_diag.gd`
   - 查看报告中腿部骨骼是否标记为 `⚠️ SCALE_3D` 或 `BLEND_SHAPE`
   - 如果是，该 GLB 必须立即拒收；旋转动画的轨道必须以 `ROTATION_3D` 正确导出

2. **定位导出源头**：
   - 检查该模型自己的 `generate_voxel_<model_id>.py` 中骨架、显式 `parts_by_bone` 和模型专属动作
   - 可复用动作或 glTF 导出机械的问题在 `voxel_character_rig.py` 修正，然后仅重新导出当前模型
   - 非人形骨架和绑定必须在该模型的生成器中修正，不得引入通用生物骨架或注册表

3. **检查动画名映射**：
   - `CombatSlashAnimator.player_animation_name(weapon)` 按武器类型映射动画名
   - 缺失动画会回退到 `"slash"`，导致攻击外观相同

4. **修复方法**：
   - 轨道损坏：在 Blender 生成/导出源头修正，重新导出后运行轨道诊断和 gdUnit 验收
   - 缺失共享动作：在 `voxel_character_rig.py` 的共享动作写入机械中补齐，再只运行该模型的 `generate_voxel_<model_id>.py`
   - 缺失专属动作、骨骼或绑定：仅在该模型生成器中补齐
   - 禁止用任何运行时脚本改写损坏轨道

## 5. 骨骼颠倒排查

### 症状
- 模型整体上下颠倒（头朝下、脚朝上）
- 部分骨骼朝向错误（手臂朝下）
- 左右镜像（Hand.R 在左侧）

### 排查步骤

1. **运行骨骼朝向诊断**：
   ```bash
   "D:/123/Godot_v4.7-stable_mono_win64.exe" --headless --path "D:/123/Lantern Tavern" \
     --script res://tools/skeleton_facing_diag.gd -- --rig=voxel_goblin_32px_rig.glb
   ```

2. **检查根节点朝向修正**：
   - 人形/史莱姆：期望 180° Y 旋转
   - 鼠/龙：期望 -90° Y 旋转
   - 报告中 `【1】根节点朝向修正检查` 会标注是否找到

3. **检查骨骼局部 Y 轴**：
   - 报告 `【2】骨骼 Rest Pose` 会标注 `⚠️ 局部Y轴朝下` 的骨骼
   - 垂直骨骼（Root/Pelvis/Torso/Head）的 Y 轴应朝上

4. **检查关键骨骼全局位置**：
   - Head 全局 Y 应 > 0.5m
   - Foot 全局 Y 应 < 0.3m
   - Hand.R/L 应在 X 轴两侧

5. **常见根因与修复**：

   | 根因 | 症状 | 修复 |
   |------|------|------|
   | 专用生成器的 LowerLeg rest pose 或绑定错误 | 小腿骨骼上下颠倒 | 修正该模型的骨骼定义与显式 `parts_by_bone`，然后只重导该模型 |
   | 朝向修正缺失 | 模型背对玩家 | 检查 `facing_rot_z` 配置 |
   | 场景覆盖了根节点旋转 | 预览正确但游戏内错 | 检查 .tscn 中 character 节点 transform |
   | PhysicalBoneSimulator3D body_offset 错误 | 死亡布娃娃骨骼乱 | 重新生成 PhysicalBone 配置 |

## 6. 朝向错误排查

### 症状
- 模型正面朝向错误（背对玩家/侧对玩家）
- 攻击方向与视觉朝向不一致

### 排查步骤

1. **确认朝向修正配置**：
   - 检查该模型自己的 `generate_voxel_<model_id>.py` 与导出根节点，朝向必须由单模型入口明确验证
   - 非人形使用该模型自己的专属骨架与朝向设计，禁止共享模型注册表

2. **检查场景是否覆盖旋转**：
   - `goblin.tscn` 中 `character` 节点的 transform 应为默认值
   - 如果场景设置了 `rotation_degrees`，会覆盖 _rig.glb 的朝向修正

3. **检查 BoneAttachment3D 朝向**：
   - `WeaponPlaceholder` 的 transform 决定武器朝向
   - `goblin.tscn` 中有具体的 `WeaponPlaceholder` transform 值

4. **验证方法**：
   - 运行 `test_humanoid_rig_glbs_have_facing_rotation` 测试
   - 使用 `_skeleton_anim_capture.gd` 截取动画中间帧查看朝向

## 7. 武器位置异常排查

### 症状
- 武器漂浮在空中或穿入身体
- 武器朝向错误（刀刃朝后）
- 攻击弧线与骨骼动画不同步

### 排查步骤

1. **检查 BoneAttachment3D**：
   - `bone_name` 应为 `"Hand.R"`（武器）或 `"Hand.L"`（盾）
   - 报告阶段4 会列出所有 BoneAttachment3D

2. **检查 WeaponPlaceholder transform**：
   - 场景 .tscn 中 `WeaponPlaceholder` 有具体的 transform 值
   - 此 transform 定义武器相对于骨骼的偏移

3. **检查 CombatSlashAnimator 弧线**：
   - `apply_weapon_arc()` 在骨骼动画之上叠加程序化弧线
   - 弧线参数：`ARC_ROLL_RAD=0.42`, `ARC_YAW_RAD=0.24`, `ARC_FORWARD_OFFSET=0.08`
   - 预览工具不会应用此弧线

4. **修复方法**：
   - 调整 .tscn 中 `WeaponPlaceholder` 的 transform
   - 不要修改 `CombatSlashAnimator` 的弧线参数（影响所有武器）

## 11. ⚠️ Headless 限制

以下内容在 headless 模式下无法验证，需在 Godot 编辑器或非 headless 运行时检查：

- **VoxelLightingAdapter 材质适配**：headless 跳过（S3TC 纹理无 GPU 无法加载）
- **SubViewport 渲染截图**：headless 无 GPU 渲染
- **实际材质外观**：需非 headless 运行 `voxel_prop_material_render_preview.gd`
- **光照/阴影效果**：需在编辑器中查看

headless 诊断工具只能验证**数据层**（骨骼名、动画名、轨道类型、变换矩阵），
不能验证**视觉层**（材质颜色、光照效果、渲染输出）。
