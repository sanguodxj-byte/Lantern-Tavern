# 02 - 酒馆与顾客系统

## 1. 酒馆的中立定位
酒馆并不是由什么“神圣结界”或“无敌光环”保护的安全区。在充满杀戮的地下城深处，它之所以能维持中立和安全，全靠以下两点：
1.  **老主顾的武力震慑**：一些在地下城极具威望和实力的早期 NPC，不允许有人在他们享受美酒的地方惹是生非。
2.  **玩家的绝对中立**：玩家不偏袒任何一方。无论是浑身是血的魔物，还是背着魔物头颅的人类冒险者，只要在店里交钱、遵守规矩，就能享有一杯麦酒的和平。

## 2. 角色类型划分：任务 NPC 与 普通顾客

地下城中登门的访客被严格划分为两类，它们在游戏循环中承担着完全不同的职责：

```
                           ┌────────────────────────┐
                           │    酒馆访客 (Visitors)  │
                           └───────────┬────────────┘
                                       │
         ┌─────────────────────────────┴─────────────────────────────┐
         ▼                                                           ▼
┌─────────────────────────────────┐                         ┌─────────────────────────────────┐
│     任务 NPC (Quest NPCs)       │                         │    普通顾客 (Regular Customers)  │
├─────────────────────────────────┤                         ├─────────────────────────────────┤
│ * 数量有限，拥有独立姓名与故事   │                         │ * 数量庞大，代表各族大众群体     │
│ * 专门用于发布任务与推动剧情     │                         │ * 人类用金币购买，魔物喝爽倾囊赠予│
│ * 完成其委托可解锁新酒谱与区域   │                         │ * 提供酒馆的日常流水与装备来源   │
│ * 拥有深度的隐性个人关系树       │                         │ * 态度与消费意愿受势力声望影响   │
└─────────────────────────────────┘                         └─────────────────────────────────┘
```

### 2.1 任务 NPC (Quest NPCs)
*   **角色特征**：数量有限，具有独特的立绘、姓名、背景故事与固定剧情。
*   **登门与常驻逻辑 (核心机制)**：
    *   **剧情进度触发**：任务 NPC 的出现不受随机概率控制。**只要玩家的剧情进度达到特定门槛**（例如：成功通关森林深处、酒馆升级到 Lv3、或完成前置主线任务），对应 NPC 就会必定在当晚主动登门，向玩家提出要求。
    *   **独立生活与非站桩逻辑**：任务 NPC 拥有独立的地下城生存轨迹，不是呆坐在固定椅子上的“站桩怪”。一旦他们向玩家提出了要求，**在玩家成功酿造出目标酒水并可以交付之前，该 NPC 绝不在酒馆干等**——他们会离开酒馆去处理自己的事情。
    *   **特定节点登门**：直到玩家在酒馆后台确认“任务酒已酿好并可以交付”，或者剧情推进到特定阶段，该 NPC 才会于当晚夜间营业阶段再次登门，取酒、结算并推进后续剧情。
*   **核心功能**：**专门发布任务与委托**。
    *   他们不参与日常的“一杯酒付多少钱”的普通消费买卖结算。
    *   玩家通过完成他们的专属任务，来获取特殊的**高适应性种族酒谱配方**、**稀有风味材料（或从人类NPC处获取大额金币、从野外遗迹搜刮获得金币）**或招募其成为**员工**。
*   **关系发展**：拥有隐性的个人关系树。关系升级会触发深度的个人剧情，并受“死亡重生失忆”的负面规则影响。

### 2.2 普通顾客 (Regular Customers)
*   **运行时动态生成机制 (Runtime Generation)**：
    *   在夜晚营业开始时，系统会动态实例化一批普通顾客。每个普通顾客在生成时，底层都会被赋予一个**独特的名字**，以及**独特的口味倾向**。
*   **“脸盲姓名锁”机制 (Name Unlocking via Familiarity)**：
    *   **不认识 (敌视/戒备/中立)**：当玩家与该怪物个体不熟悉时，玩家是**无法查看或分辨其名字**的。所有同一物种的怪异在玩家 UI 和对话框中表现为完全一样的大众脸，统一显示为种族泛指（如 `“哥布林”`、`“牛头人”`）。
        *   **熟识解锁本名并录入常客名册 (熟悉/友好 - 常客上限与轮换)**：只有当玩家通过持续为其提供其喜爱的酒水、使该魔物个体的个人好感度提升至“熟悉”及以上时，玩家才会**解锁并记下其真实本名**（如 UI 中的 `“哥布林”` 变化为 `“格鲁姆”`），此顾客也将正式录入酒馆的“常客名册”。
            *   **名册上限**：酒馆的常客名册全局容量上限为 **$25$ 名**（5大怪物种族哥布林、牛头人、幽灵、精灵、巨人各占最多 5 席）。
            *   **客流轮换机制**：每晚来店的顾客流中，系统有 $30\%$ 的概率直接从已录入常客名册的熟面孔中挑选实例化；另外 $70\%$ 为未解锁本名的随机生面孔。若常客名册席位已满，新解锁本名的顾客会自动替换掉该族常客池中“好感度最低且最久未来店”的常客。
*   **“种族趋同”与“个体微调”口味体系**：
    *   **种族口味趋同**：同一物种 of 魔物，其口味偏好在大体上是统一的（如：所有哥布林都喜欢 `[腐败, 甜美]`，讨厌 `[苦涩]`）。
    *   **个体风味微调**：在种族趋同的基础上，每个运行时生成的怪物个体都会携带微小的**个性化口味偏差**（例如：格鲁姆这只哥布林除种族偏好外，个体还微弱地喜欢 `[咸度: +1]`、讨厌 `[辣口: -1]`）。
    *   **玩法联动**：玩家做普适大宗流水生意时，只需酿造符合种族趋同偏好的酒水；不过若想重点培养某个“熟人”，则需在交互中留心观察、记录并迎合该个体的微调偏好，以获取最大好感增幅。
*   **双轨支付与赠予机制 (核心结算机制)**：
    *   **人类顾客 (自定义标价与偏离度评估制)**：
        *   **定价权归玩家**：人类拥有成熟的商业契约和货币共识，玩家可以通过酒馆菜单自由给酒水设定自定义标价 $P_{menu}$。
        *   **偏离度价格匹配评估**：以酒水的基础标准价格 $P_{base}$ 为基准，人类顾客点单时对标价 $P_{menu}$ 进行合理性评估判定：
            *   **实惠赞赏 ($P_{menu} \le P_{base}$)**：玩家获得 $P_{menu}$ 金币。该人类顾客对玩家的好感度 $+1$。
            *   **合理接受 ($P_{base} < P_{menu} \le 1.3 \times P_{base}$)**：人类会微微抱怨被宰，但仍正常足额支付 $P_{menu}$ 金币。
            *   **昂贵抱怨 ($1.3 \times P_{base} < P_{menu} \le 1.6 \times P_{base}$)**：人类被迫支付 $P_{menu}$ 金币，但会严重抱怨，导致酒馆当晚“全局传闻声望”被扣除 $2$ 点。
            *   **拒付离席 ($P_{menu} > 1.6 \times P_{base}$)**：价格严重超出其底线，人类顾客直接摔杯拒付（获得 $0$ 金币）并愤怒离席，导致全局声望暴跌 $15$ 点。
    *   **怪物顾客 (倾囊赠予制)**：
        *   **无货币与经济共识**：魔物不具备任何经济体系，也完全不理解金币在人类社会的商业交易价值。它们收集金币纯粹是因为金币在地下城属于“闪亮亮且坚硬的金属圆片（玩具）”，或是因为它们知道人类（玩家）特别喜欢这种黄灿灿的亮铁片。
        *   **喝爽直接倾囊**：怪物给亮铁片（金币）的机制并非“买卖付款”，而是“赠予”。当怪物喝酒喝爽了（满意度达到对应阈值），它们出于狂热与开心，会将自己随身携带的所有“亮铁片玩具（金币）”甚至身上的防具、武器一股脑全部拍在桌上送给老板。如果酒极不合口味，它们会感到愤怒而拒绝赠送任何东西，甚至摔杯子离去。
        *   **机制引导**：经营怪物顾客的关键在于“调制出极致契合其种族与个体微调偏好的美酒”以骗取其全部的亮铁片与随身宝贝，并设法“提高势力声望”吸引拥有更多高阶亮铁片和史诗级玩具的魔物精英。
    *   **抵账与赠予机制**：当普通顾客（无论是人类钱不够触发抵账，还是魔物因未携带亮铁片而触发赠予）结算时，会将身上的常规武器、防具或材料留下给老板。
*   **口耳相传的情报与事件系统 (Word of Mouth System)**：
    *   **唯一信息源**：游戏内**完全没有任何系统提示弹窗、小黑板天气预报或静态日志通知**。
    *   **对话承载万物**：所有的地下城突发事件（如寒流爆发、火山异变）、明天顾客的口味需求波动、新区域高级材料的坐标采集指引、以及主支线故事的推进，**均完全通过酒馆内 NPC 和普通顾客的嘴里说出来（头顶闲聊台词与对话气泡）**。
    *   **收集与规划**：玩家在柜台营业和与顾客对话时，必须及时点击顾客们聊天浮出的悬浮情报气泡（例如：*“隔壁盲鱼干最近腥臭难当，不过哥布林那边倒是开出大价钱收它……”*），并据此在深夜规划明天该去哪里探索、该在缸里酿造什么酒。
    *   **口耳相传情报触发决策树 (Gossip Trigger Decision Tree)**：
        为了确保情报流出的确定性与随机性平衡，系统在顾客喝完酒并满意度结算后，执行以下决策树来决定其闲聊气泡所吐露的内容：
        ```mermaid
        graph TD
            Start[顾客喝完这杯酒并顺利结算] --> CheckS{满意度 S >= 5 ?}
            CheckS -->|否| NoClue[【日常普通闲聊】<br>仅说一些无营养的种族设定口癖台词]
            CheckS -->|是| CheckActivePlot{当前是否有未完成的剧情/主线任务?}
            
            %% 任务推进分支
            CheckActivePlot -->|是| IsQuestNPC{该顾客是本任务的主角 NPC?}
            IsQuestNPC -->|是| NPCClue[【核心故事台词】<br>直接告知当前剧情背景、下一步需求、并明确指引核心材料位置]
            IsQuestNPC -->|否| GossipClue[【剧情周边八卦】<br>普通怪物碎碎念爆料该 NPC 相关的线索或当前传闻]
            
            %% 日常情报分支
            CheckActivePlot -->|否| RollGossip{随机事件 Roll 点判定 (30% 概率)}
            RollGossip -->|失败 70%| ThankTavern[【致谢口头禅】<br>单纯吹捧酒馆酒好，无实用信息]
            RollGossip -->|成功 30%| SelectGossip[【透露实用情报】<br>随机从日常情报池挑选并吐露一条:<br>1. 某个生态锁异常 (哪里怪物聚集变多)<br>2. 特殊材料位置 (哪里发现了稀有采集点)<br>3. 外部异动 (天气变化或未来几天某种口味供求大涨)]
        ```
    *   **信息流转逻辑**：
        1.  **满意度门槛**：只有当顾客对酒水满意度极高（$S \ge 5$，即喝爽了），才会向玩家吐露有用情报。这建立了“酒酿得好才能获得地底世界情报”的强反馈环。
        2.  **剧情锁定**：一旦系统判定有激活的任务尚未完成，普通顾客的情报池会优先偏向输出与该任务有关的侧面八卦，以保证玩家不会在探索中因为“没有小黑板日志”而彻底迷失方向。

## 3. 2D 像素俯视角下的经营具体视觉玩法 (Tavern Gameplay Mechanics)

由于本作酒馆**只售卖自定义美酒，不提供任何餐食**，因此所有的核心买卖交易、点单与结算，**全部在“吧台柜台”这一物理网格处完成**。这极大地简化了玩家在夜晚营业时的操作压力。

### 3.1 画面区域划分 (Visual Layout)
在酒馆经营的俯视网格中，整个酒馆的核心视点与交互被高度收拢于吧台区域，画面被清晰划分为：
1.  **吧台操作区（主交互区）**：老板（玩家）的工作领地。柜台后一字排开 4 口**装酒的木桶**。柜台前是顾客排队和坐着饮酒的吧台凳。
2.  **氛围慢饮区（散座背景）**：散布在吧台周围的桌椅客座仅作为烘托温馨氛围的视觉背景（无任何游戏数值与机制限制，怪物在此活动坐下属于纯视觉表现）。**所有与玩家有关的点单、饮酒、结算与赠予、冒情报气泡的实质交互，全部在柜台吧台前完成**，玩家不需要离开吧台去其他区域。
3.  **地窖与后院（屏幕右侧）**：连通玩家的材料仓库与二手装备货架。

### 3.2 营业期玩家操作的“渐进式自动化”演进 (Daily Operations & Staff Progression)
夜晚营业时，玩家操作被简化为纯粹的“酒保打酒”与“看戏倾听”：

#### 3.2.1 游戏前期 (酒馆 Lv1)：极简的柜台酒保体验
*   **经营现状**：只有 1-2 个酒桶位，客流极低，没有雇佣打酒员工。
*   **玩家操作动作 (仅限柜台后微幅移动)**：
    1.  **柜台点单**：顾客走到柜台前指明喜好（如魔物想要苦涩/麦香味酒）。
    2.  **选桶打酒**：玩家控制老板在吧台后左右移动一两格，移动到对应的酒桶龙头后，交互打酒并递给面前的顾客。
    3.  **收钱与点击气泡**：顾客接过酒当场在柜台喝掉，金币自动吸入钱包。喝爽的顾客头顶冒出黄色八卦气泡，**玩家用鼠标直接在屏幕上点击气泡**即可收集，无需控制老板移动过去偷听。
    *   *设计意图*：去除了满店跑堂和洗杯子的劳累，玩家只需在柜台后选择正确的龙头倒酒，看着顾客大口喝酒、金币暴涨、气泡浮现。操作极少，反馈极大，非常治愈。

#### 3.2.2 游戏中期：野外事件招募店员与自动打酒
*   **招募店员**：玩家在白天野外探索遭遇特定事件并招募到店员（精灵或牛头人）。这属于纯探索/剧情事件直接招募获取，与酒馆物理等级脱钩。
*   **自动打酒**：招募到店员后，夜晚营业他们将自动进驻大堂柜台，接管普通顾客的点单、打酒和收钱工作。
*   **玩家职责**：玩家彻底不用动手，只负责在柜台接待特定主动登门找你说话的任务 NPC，倾听他们的主线故事。

#### 3.2.3 游戏后期 (酒馆 Lv4+ - 完全自动化与终极治愈)
*   **全自动看戏**：柜台打酒全自动流转。玩家在营业期间只需放松地听着吉他民谣，用鼠标一键收集空中浮现的情报气泡，享受金币清脆落袋和怪物喝爽冒爱心的纯治愈画面。

## 4. 玩家身份演进系统 (Player Status Progression)
玩家的身份随着在探索 and 经营中的表现，在地下城社会中发生隐性的、渐进式的演变：

*   **无名冒险者**：地下城居民默认视你为掠夺者/猎人，野外遭遇高度敌对。
*   **开酒馆的人类**：开始引起周围部落的注意，敌意减弱，不过普通顾客登门时依然充满戒备。
*   **酒馆老板**：各势力认可酒馆的缓冲区地位，普通顾客流源源不断，任务 NPC 开始主动登门发布委托。
*   **地下城商人**：成为不可或缺的物资流转节点，普通顾客喝爽了会慷慨赠送其随身的高阶装备，可与势力高层（领主 NPC）直接对话。
*   **地下城的一份子**：身份彻底融入生态，一举一动都将重塑地下城的势力格局。

## 5. NPC 记忆系统与三层关系模型

### 5.1 双轨关系与好感度机制 (核心社交规则)
*   **人类顾客 (无好感系统)**：
    *   **人类完全不具备任何“好感度”系统**。人类与玩家的关系是纯粹的商业买卖契约，他们只关心酒水口味倾向对胃口和标价的合理性，不记录个体情感纽带。
*   **怪物顾客 (酒水馈赠涨好感)**：
    *   **喝酒即涨好感度**：无论是剧情 NPC 还是普通怪物顾客，**只要在酒馆喝到了酒，就必定会增加其好感度**（哪怕他们倾囊后只赠予了极少亮铁片，也会因为获得酒精招待而对玩家产生感激）。
*   **怪物好感的“个体向群体”传递机制**：
    *   怪物关系分为**“个体好感度”**（针对具体有名字的任务 NPC，或被系统记录特征 of 普通顾客个体）与**“群体/势力关系”**（例如整个哥布林部族的态度）。
    *   **间接折算**：当个体的怪物对玩家好感度上升时，其好感增幅会按比例（如：个体好感的 10%）间接折算，累积到其所属的群体/势力关系中。善待每一个普通怪物，是拉拢整个怪物部落的根本途径。
    *   **关系影响**：势力关系的高低，直接决定了该种族普通顾客的每日登门流量、以及点高档酒的支付钱包上限。此外，**当该种族势力声望达到极高阶段（如友好、信任）时，该族的普通顾客在喝爽后，有很高概率会在赠予亮铁片之余，额外主动向玩家赠礼，送上一份该族特有的稀有材料**。这让声望收益直接反哺到酒馆的材料储备上，极其直观并无需增加野外循环的开发负担。

### 5.2 记忆标签系统 (Memory Tag System)
系统会动态记录玩家与 NPC 交互时的重要行为，并为其打上**印象标签**。这些标签是决定 NPC 态度转变的直接依据：
*   `曾交流`：建立社交往来的开始。
*   `曾帮助`：在探索中解救了被围困的 NPC，或者在其极度干渴时赠予了饮品。
*   `曾击败`：在战斗中将该 NPC 击败并选择放生。
*   `被击杀 (无仇恨)`：即使玩家在野外将该魔物 NPC 击杀，由于其“地底复生规则”，**被杀不会生成任何负面/仇恨标签**。魔物默认胜败乃地底常事。
*   `失信者`：在交易中违约或欺骗，这会切实恶化个人和势力关系，生成强烈的负面标签。
*   `原主人`：该怪物顾客在酒馆里用随身装备作为赠礼留给玩家。若玩家佩戴该装备与其对话，或将该装备在酒馆里交易/归还给他，会触发趣味的彩蛋闲聊台词（仅用作氛围渲染，无数值与属性影响）。

### 5.3 三层关系判定
```
┌──────────────────────────────────────────────┐
│             第一层：个人关系 (仅限任务NPC)     │  <-- 深度情感与故事解锁
├──────────────────────────────────────────────┤
│             第二层：势力关系 (针对普通顾客)   │  <-- 决定日常客流、消费力与野外警戒度
├──────────────────────────────────────────────┤
│             第三层：传闻声望 (全局事记)      │  <-- 影响力(如: 玩家击败了森林领主)
└──────────────────────────────────────────────┘
```
*   普通顾客的交互主要读取**第二层：势力关系**。
*   任务 NPC 的交互则主要读取**第一层：个人关系**。
