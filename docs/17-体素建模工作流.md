# 体素建模工作流

本项目体素模型固定比例尺：`1m = 32px`，也就是 `1px = 1/32m`。所有模型先用像素尺寸设计，再由代码转换为米制单位。

## 1. 建模前尺寸表

每个物体先写清楚目标尺寸：

- 桌子：长 `64px`，宽 `34px`，高 `26px`。
- 小型壁挂火把静态本体：高不超过 `14px`，火焰为动态粒子，不计入静态本体。
- 蜡烛、柄、铆钉等小部件优先使用 `1px / 3px / 5px` 奇数宽度，避免中心线偏半格。

禁止先凭感觉摆米制坐标。必须先确认像素尺寸，再换算到 Godot 单位。

## 2. 附着规则

所有静态体素盒必须组成一个整体：

- 允许面接触。
- 允许有意重叠。
- 禁止只靠边/角接触。
- 禁止悬空分离。
- 禁止火把柄、杯口、墙托等视觉上断开。

动态火焰、声音、灯光可以不是静态体素盒，但它们的位置必须贴合静态承载部件。

## 3. 自动验证

修改体素模型后至少运行：

```bash
"D:/123/Godot_v4.7-stable_mono_win64.exe" --headless -s tests/gdunit4_runner.gd -- --ignoreHeadlessMode -a tests/gdunit/voxel_prop_scene_test.gd
```

该测试会检查：

- 所有盒体尺寸按 `1px = 1/32m` 对齐。
- 小型细节使用奇数像素宽度。
- 关键物品比例尺正确。
- 所有静态体素盒是单一附着组件。
- 火焰不被建进静态模型。

## 4. 三视图验证

修改或新增体素模型后生成三视图：

```bash
"D:/123/Godot_v4.7-stable_mono_win64.exe" --headless -s tools/voxel_prop_three_view_capture.gd
```

输出目录：

```text
reports/props_preview/
```

必须检查每个模型的：

- `front`
- `side`
- `top`

三视图里必须确认比例正确、部件附着正确、没有错位、悬空、穿帮、动态效果位置错误。

## 5. 提交流程

1. 写像素尺寸表。
2. 建静态体素主体。
3. 检查每个部件是否面接触或重叠。
4. 添加动态效果和光源。
5. 运行 `voxel_prop_scene_test.gd`。
6. 生成三视图截图。
7. 人工检查三视图后再继续。
