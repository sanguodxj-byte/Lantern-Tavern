# 体素建模工作流

本项目体素模型固定比例尺：`1m = 32px`，也就是 `1px = 1/32m`。所有模型先用像素尺寸设计，再由代码转换为米制单位。

## 1. 建模前尺寸表

每个物体先写清楚目标尺寸：

- 桌子：长 `64px`，宽 `34px`，高 `26px`。
- 小型壁挂火把静态本体：高不超过 `14px`，火焰为动态粒子，不计入静态本体。
- 蜡烛、柄、铆钉等小部件优先使用 `1px / 3px / 5px` 奇数宽度，避免中心线偏半格。

禁止先凭感觉摆米制坐标。必须先确认像素尺寸，再换算到 Godot 单位。

角色生成器的尺寸表必须写出最终 `width x height x depth` 包络，并由该角色的
gdUnit 测试按像素容差锁定。部件数量不能替代包络验证；主体变窄、变矮后增加贴片，
仍然属于体量退化。

### 1.1 默认对称原则

- 除非设计上需要凸显明确的非对称语义（例如衣物、装备、伤痕、装饰或姿态），模型主体与成对部件应围绕中心轴大致镜像。
- 任何非对称设计都必须能够说明其视觉或玩法含义；禁止无意义的单侧突起、贴片、色块或尺寸差异。
- 非对称只应用于承载该语义的局部部件，不能顺带破坏头部、躯干、四肢等其余主体的对称关系。

### 1.2 角色美术方向

角色采用《Barony》式作者化体素艺术，并以项目 S 级岩石魔像作为内部质量参照：

- 主体由成组体素形成阶梯状、不规则且一眼可辨的轮廓，不是光滑大方盒。
- 约八成视觉体量应由头、胸腔、骨盆、肢体或生物核心等主形体承担；微型体素只表达眼睛、裂纹、牙齿、徽记和材质边界。
- 每种主要材质使用有目的的高光/中间色/阴影色阶，并通过真实深度层而不是大量薄贴片建立明暗。
- 正面、侧面、顶面都必须能读出身份锚点。只有正面贴图式细节、侧面完全扁平的模型不合格。
- 禁止共享人形身体、族群身体或 silhouette 模板。共享代码只能提供立方体原语、面接触定位、校验、导出、渲染和动画机械。

旧死灵领主只可参考“王冠、骷髅面、暗紫长袍、青色灵魂光、权杖”的身份语言；
旧拓扑、旧生成脚本和所有非 S 旧 GLB 均不得复用。

## 2. 附着规则

所有静态体素盒必须组成一个整体：

- **只允许面接触**（恰好一个轴贴齐，另外两个轴有正重叠）。
- **禁止正体积重叠**（三轴同时正重叠）——同材质或共面跨材质都会深度闪烁（z-fighting）。
- **附着连通性只认面接触**：正体积重叠**不算**附着，也**不能**用来「粘」部件。
- 装饰层（高光/阴影/胸甲贴片/披风等）必须贴在主体**外侧**，用 `host_half + plate_half` 放置中心，禁止负偏移陷入主体。
- 确实需要穿插时，必须写清楚渲染层级/深度偏移方案，并补角色/道具 overlap 回归测试白名单说明。
- 禁止只靠边/角接触。
- 禁止悬空分离。
- 禁止火把柄、杯口、墙托等视觉上断开。

动态火焰、声音、灯光可以不是静态体素盒，但它们的位置必须贴合静态承载部件。

### 2.1 装饰外贴公式（像素单位）

宿主盒中心 `C`、尺寸 `S`，贴片厚度 `T` 沿轴 `axis` 外侧：

```
plate_center[axis] = C[axis] ± (S[axis]/2 + T/2)
```

生成器共享工具：`tools/voxel_overlap_guard.py` 的 `exterior_plate_center()`。

## 3. 自动验证

修改体素模型后至少运行：

```bash
# 道具场景附着 + 比例
"D:/123/Godot_v4.7-stable_mono_win64.exe" --headless -s tests/gdunit4_runner.gd -- --ignoreHeadlessMode -a tests/gdunit/voxel_prop_scene_test.gd

# 道具正体积重叠（z-fighting）
"D:/123/Godot_v4.7-stable_mono_win64.exe" --headless -s tests/gdunit4_runner.gd -- --ignoreHeadlessMode -a tests/gdunit/voxel_overlap_scan_test.gd

# 角色/怪物 GLB 正体积重叠（z-fighting）——角色管线强制
"D:/123/Godot_v4.7-stable_mono_win64.exe" --headless -s tests/gdunit4_runner.gd -- --ignoreHeadlessMode -a tests/gdunit/character_voxel_overlap_test.gd
```

这些测试会检查：

- 所有盒体尺寸按 `1px = 1/32m` 对齐。
- 关键物品比例尺正确。
- 所有静态体素盒是**单一面接触连通**组件。
- 道具与角色均无「同材质正体积重叠」与「跨材质共面重叠」。
- 火焰不被建进静态模型。

### 3.1 生成器出口硬断言

Blender 生成脚本在 `export` 前必须调用：

```python
from voxel_overlap_guard import assert_parts_voxel_assembly_valid
assert_parts_voxel_assembly_valid(parts, label="creature_id")  # no overlap + face-connected
```

断言失败 → **禁止写出 GLB**。每个角色/怪物必须由自己的
`tools/generate_voxel_<model_id>.py` 调用该断言；道具也必须由单一资产生成器调用。
旧的档位、名单或目录批量生成器均为禁止入口，不得用于重新导出资产。

## 4. 三视图验证

修改或新增体素模型后生成三视图：

```bash
"D:/123/Godot_v4.7-stable_mono_win64.exe" --headless --path "D:/123/Lantern Tavern" --script res://tools/voxel_prop_three_view_capture.gd -- --asset=<model_id>
```

模型验收必须使用精确的 `--asset=<model_id>`。无选择器、`--monsters-only`、档位、
通配符、多 ID 或目录循环只能用于只读诊断，不能作为该模型已完成验证的证据。

角色/怪物真 3D `front/side/top/preview` 必须由该模型自己的
`tools/generate_voxel_<model_id>.py` 产出。不得另用接受逗号列表、全目录扫描或多目标循环的
预览入口替代单模型生成器内的验证输出。

真 3D 输出固定命名为
`voxel_<model_id>_render_{preview,front,side,top}.png`；结构投影继续使用
`voxel_<model_id>_{front,side,top}.png`。两类捕获禁止同名，后运行者不得覆盖前者。

输出目录：

```text
reports/props_preview/          # 道具结构/预览
reports/characters_preview/     # 角色三视图 + preview
```

必须检查每个模型的 `front` / `side` / `top`：

- 比例正确、部件附着正确
- 无错位、悬空、穿帮
- **无可见 z-fighting 闪烁带**
- 动态效果位置正确

> **禁止**用 ~2KB 结构 2D 投影冒充材质/光照验收。结构投影只用于拓扑；生产观感必须用真 3D 渲染或引擎截图。

## 5. 真·重做门槛（remake 定义）

以下任一条不满足，只能标为「模板增量 / 贴片改皮」，**不得**称为 remake 或 A/S 旗舰：

1. 独立 silhouette 像素尺寸表（头身比、肩宽、体长），不是 48px 人形底板换色。
2. 主形体占据目标包络并形成专用轮廓；mesh 数量不是质量门槛，微贴片不得替代主体体量。
3. 导出前 `voxel_overlap_guard` 全绿。
4. `character_voxel_overlap_test` 全绿。
5. 真 3D `front/side/top/preview` 已更新且非结构投影。

所有非 S 旧模型及其旧生成器已判定为污染。重建某只模型时必须从空白的
`tools/generate_voxel_<model_id>.py` 开始，在该文件中独立定义 silhouette、尺寸、材质、
骨骼分组和语义部件；禁止恢复历史玩家、人形、creature 或 roguelike 批量入口。

## 6. 提交流程

1. 只选择一个模型 ID，并只打开它的 `tools/generate_voxel_<model_id>.py`。
2. 写该模型自己的像素尺寸表。
3. 建静态体素主体。
4. 检查每个部件是否**仅面接触**，且无正体积重叠。
5. 装饰层用外贴公式放置。
6. 生成器内 `assert_parts_no_positive_volume_overlap`。
7. 运行该模型专用测试及相应 overlap 测试。
8. 使用 `--asset=<model_id>` 生成结构三视图，并生成该模型的真 3D 预览。
9. 人工检查全部视图后，才能结束该模型并开始下一只。
