# 20 - 被动机制树（构筑骨架 / Passive Mechanic Tree）

> **定位**：本文档是「符文构筑深度」的**骨架层**。
> - **乘数层**在 `19-符文系统（构筑深度）.md`：符文（含 A/B 两类）负责"同一机制如何被放大 / 变形"。
> - **骨架层在本文件**：双轨领悟被动树负责"武器**获得什么机制**"——它是 build 的骨架，决定一个 build 的**本质差异**；符文只是血肉，放大已存在骨架上的机制。
>
> **关键模型（v3 统一裁定）**：**武器层 与 流派层 是两个完全解耦的独立变量。**
> - **武器层 (Weapon)**：具体的武器装备实体（单手剑、双手大剑、双手斧、战锤、长枪、长弓、轻弩、附魔法杖、魔导书、盾牌等）——决定基础攻击形态与武器固有机制。
> - **流派层 (Style)**：纯粹由**手部握持姿态 / 装备占用形态**决定（**单手** `ONE_HAND`、**持盾** `ONE_HAND_SHIELD`、**双手** `TWO_HAND`、**双持** `DUAL_WIELD`、**徒手** `UNARMED`、**远程** `RANGED`、**法系** `SPELL`），**与具体使用的是什么武器完全无关**。
> - 一个 Build 的最终机制 = **装备武器固有机制 (Weapon Passives) + 握持流派机制 (Style Passives)**；B 类符文放大/升级的是这二者合成后的结果。


---

## §1 双层模型（武器层 × 流派层）

| 层 | 决定什么 | 来源 | 数量 | 可组合性 |
|---|---|---|---|---|
| **武器层** | 拿什么武器、单手/双手、能否配盾、基础攻击 | `skill_data.gd` 武器媒介定义 | 10 种武器 | 每把武器有固定握持属性 |
**组合规则（全局统一）**
- **握持流派 (Style)**：由玩家手部装备姿态（单手、持盾、双手、双持、徒手、远程、法系）自动激活，**不绑定任何具体武器实体**。
- **武器被动 (Weapon)**：任何单手/双手武器只要装备上，即提供其固有的武器被动（如单手剑 `parry_lean`、双手大剑 `cleave`、双手斧 `execute`）。
- 玩家的完整 Build 效果由【手部姿态激活的流派被动】与【所持具体武器的固有被动】叠加合成。

**解锁链路（写实自 `skill_data.gd`）**
- 武器层按 `proficiency` 熟练度解锁基础攻击与固有机制。
- 流派层按双轨领悟阶梯（proficiency + 主属性门槛）解锁（见 `15` §1.1）：
  - T1：`proficiency ≥ Lv3` 且 主攻属性 `≥ 15`
  - T2：`proficiency ≥ Lv8` 且 主攻属性 `≥ 35`
  - T3：`proficiency ≥ Lv15` 且 主攻属性 `≥ 70`
- 持盾流派额外要求「装备盾牌」才生效（流派机制 = 条件授予）。
- **关键**：武器层与流派层均为 **meta 持久**（双轨领悟阶梯，跨 run 保留）；符文是 **run-scoped**（回酒馆丢弃）。即「机制框架常驻、符文临时放大」。

## §2 武器层（基础）

> 每把武器有**固有机制（inherent）**——这是武器本身的味道，不是完整 build。握持属性决定它能进哪些流派。

| 武器 (Weapon) | 握持 | 可否配盾 | 固有机制 id | type | 固有机制语义 |
|---|---|---|---|---|---|
| **单手剑** `ONE_HAND_SWORD` | 纯单手 / 可配盾 | 配盾✅ 纯单手无盾✅ | `parry_lean` | `guard_posture` | **唯一有「纯单手无盾」形态的武器**；也可配盾 |
| **战锤** `WAR_HAMMER` | 配盾 / 双手（无纯单手） | 配盾✅ 纯单手❌ | `impact` | `stagger_stack` | 命中叠硬直；**无纯单手形态**，要么配盾要么双手 |
| **长枪** `SPEAR` | 配盾 / 双手（无纯单手） | 配盾✅ 纯单手❌ | `reach` | `extended_reach` | 超距戳刺；**无纯单手形态**，要么配盾要么双手 |
| **双手大剑** `TWO_HAND_SWORD` | 双手 | ❌ | `cleave` | `melee_splash` | 横扫溅射 |
| **双手斧** `TWO_HAND_AXE` | 双手 | ❌ | `execute` | `execute_threshold` | 低血处决 + 破甲 |
| **长弓** `LONGBOW` | 双手(远程) | ❌ | `pierce` | `projectile_pierce` | 穿透 + 标记 |
| **轻弩** `LIGHT_CROSSBOW` | 双手(远程) | ❌ | `reload_shot` | `crossbow_reload` | **装弹射击**：每次射击后进入未装弹态，须完成装弹（约 1.2s，可被打断重来）才能再次射击，**不允许连续发射**；命中附带钉身定身 |
| **附魔法杖** `ENCHANT_WAND` | 单手(法) | ❌ | `element_infuse` | `element_infuse` | 武器附元素(燃烧/冰)，联动 ember |
| **魔导书** `GRIMOIRE` | 双手(法) | ❌ | `spell_echo` | `spell_echo` | 法术重复，联动 echo |
| **徒手** `UNARMED` | 徒手 | ❌ | `lifedrain` | `lifesteal` | 吸血 + 擒抱投掷 |

> 注：战锤 / 长枪**无纯单手形态**，只有「配盾」与「双手」两种配置，不存在"单手战锤/长枪独自挥舞"的形态。单手剑是**唯一**拥有「纯单手无盾」形态的武器。

## §3 流派层（握持姿态）

> 流派是 Build 的手部姿态身份。完全由**装备占用姿态**驱动，与具体使用的是单手剑还是战锤等武器实体无任何绑定关系。

### 3.1 握持流派被动机制授权

每个握持流派根据其姿态授予专属的流派效果，与武器固有被动叠加生成最终的 Build 效果。

### 3.2 手部握持流派机制授权表 (由手部姿态激活，不限具体武器实体)

| 握持姿态流派 (Style) | 触发/生效模式 | 授予流派机制 id | 机制语义 |
|---|---|---|---|
| **单手流派** `ONE_HAND` | 主手单手，副手空 | `duelist_stance` / `flurry` | **灵动连打与快速反应**：提供攻速、闪避加成与连击缩短 |
| **持盾流派** `ONE_HAND_SHIELD` | 主手单手，副手防盾 | `shield_bash` / `shield_allow_skill` | **盾击与稳定防御**：提供完美格挡顿击、正面免伤与持盾技能释放 |
| **双手流派** `TWO_HAND` | 双手握持武器 | `heavy_impact` / `guard_breaker` | **重压击退与破防**：增加击退冲量、蓄力减伤与对盾牌耐久压制 |
| **双持流派** `DUAL_WIELD` | 主手单手，副手单手 | `dual_flurry` / `cross_strike` | **双刃连击**：提升攻速移速，副手命中为主手增加暴击加成 |
| **徒手流派** `UNARMED` | 双手完全空置 | `flurry_combo` / `deflect` | **寸劲与借力偏转**：缩短前摇、连击必暴与偏转反弹 |
| **远程流派** `RANGED` | 握持远程弓/弩 | `distance_bonus` / `kite_step` | **远距加成与游走**：依距离提供伤害加成与命中后移速提升 |
| **法系流派** `SPELL` | 握持法杖/魔导书 | `spell_focus` / `spell_barrier` | **施法爆破与护盾**：施法概率重置 CD 与获得临时法力护盾 |

一个 Build 最终合成的机制 = **装备武器固有被动 (Weapon) + 握持流派被动 (Style)**。示例：

| 装备武器实体 (Weapon) | 触发的握持流派 (Style) | 合成机制描述 |
|---|---|---|

| **战锤** | **双手流派** `TWO_HAND` | `impact` (命中叠硬直) + `heavy_impact` (击退+2m) + `guard_breaker` (破防) |
| **战锤** | **持盾流派** `ONE_HAND_SHIELD` | `impact` (命中叠硬直) + `shield_bash` (盾击) + `firm_stance` (正面减伤) |

## §4 属性里程碑轴（数值底盘，写实自 doc15 §3.2）

6 主属性 × 3 阶，常驻生效，不依赖武器/流派。与 §2/§3 正交组合，是 build 的"数值底盘"。

| 主属性 | T1(≥5) | T2(≥15) | T3(≥30) |
|---|---|---|---|
| **力量 STR** | 震退(15%击退2m) | 重力击(暴击必击退) | 蛮力负荷(负重+15,双手+5%伤) |
| **敏捷 DEX** | 跳跃(2m跨越) | 神射手(远程命中+10%) | 穿透打击(远程无视10%物防) |
| **灵巧 AGI** | 侧垫步(10%闪避免伤) | 轻捷之行(移速+10%) | 虚实避让(闪避+6%) |
| **体质 CON** | 强健体魄(最大HP+20) | 厚实皮肤(每次伤害-2) | 复苏之息(休整回25%HP) |
| **魔力 MAG** | 魔力凝息(10%冷却清零) | 元素护壳(法伤-4) | 魔力涌流(法技暴击+8%) |
| **感知 PER** | 警觉(视野+2m,陷阱+20%) | 弱点洞察(暴击+5%) | 直觉闪避(免疫侧击/背袭加成) |

**组合示例**：
- 单手剑×持盾 + STR T3（双手+5%伤→对盾击同样生效）+ CON T1/T3（HP+20/回血）→ 稳健坦克流。
- 单手剑×迅捷剑术 + DEX T2（远程无关，但 AGI T2 移速+10%）+ PER T3（免疫背袭）→ 灵巧反击刺客流。
- 附魔法杖 + MAG T1（10%冷却清零）+ MAG T3（法技暴击+8%）→ 高频燃烧法杖流。

## §5 机制 × 符文增强映射（骨架 × 乘数）

B 类符文在 §2/§3 合成机制上做**增强(Amplify, 缩放 params)** 或 **机制加强(Upgrade)**。下表给出代表性映射（B 类池名沿用 `19` §3.1）：

| 合成机制 | 增强型 B 符文（缩放，正典名） | 机制加强型 B 符文（升级，正典名） |
|---|---|---|
| `shield_bash` 盾击 | 守护(Othala)：盾击伤害 +25% | 连锁(Nauthiz)：盾击溅射相邻 |
| `riposte` 反击 | 守护(Othala)：反击 +25% | 连锁(Nauthiz)：反击溅射 |
| `cleave` 溅射 | 灼燃(Fehu)：溅射附燃烧 | 蔓延(Hagalaz)：范围 +1m |
| `execute` 处决 | 破晓(Dagaz)：处决阈值 +5% | 生息(Berkano)：处决回血 |
| `stagger`/`impact` 硬直 | 寒霜(Isa)：每击 +0.05s 硬直 | 破晓(Dagaz)：满层转眩晕 |
| `reach`/`brace` 控距 | 蔓延(Hagalaz)：攻击距离 +0.5m | 破晓(Dagaz)：架枪反冲穿透 |
| `pierce` 穿透 | 破晓(Dagaz)：穿透 +1 | 连锁(Nauthiz)：标记连锁 2 |
| `reload_shot` 装弹射击 | 迅捷(Ehwaz)：装弹时间 -30% | 寒霜(Isa)：定身 +1s |
| `element_infuse` 附魔 | 灼燃(Fehu)：燃烧 dps×1.3 | 蔓延(Hagalaz)：燃烧蔓延 |
| `spell_echo` 回响 | 回响(Jera)：重复 +1 | 破晓(Dagaz)：回响 60%→80% |
| `lifedrain` 吸血 | 生息(Berkano)：吸血 +10% | 生息(Berkano)：处决触发回血翻倍 |

> `element_infuse` / `spell_echo` 的 B 类键（`burn_*` / `repeat_count`）**已存在于现有 `rune_data.gd` 的 `ember` / `echo`**——说明符文层早已预留机制接口，骨架层只需确保消费端路由这些 type（`19` G8）。其余机制（shield_bash/riposte/cleave/execute/stagger/reach/pierce/reload_shot/throw/lifedrain）需在 G8 补路由，否则 B 类符文"镶了没用"。

## §6 消费管线实现注记（指向 `19` G1/G8）

1. **授予**：武器固有机制存于武器定义；流派机制存于流派被动的 `weapon_mechanic` 字段（结构见 `19` §11.2），**条件流派**（如持盾）需额外判定装备状态。
2. **聚合**：`skill_runtime.gd` 的被动槽聚合「武器固有 + 已激活流派机制」，应用到**手持武器 + 装备状态**（如持盾）。一处机制可被多源叠加。
3. **分发**：战斗/装备层机制分发器按 `type` 路由到盾击/反击/燃烧/连锁/回血等处理（`19` G8）。
4. **符文叠加**：B 类符文在聚合后对其 `params` 缩放（Amplify）或注入 `mechanic_upgrade`（Upgrade），结算「先乘后加」（`19` §6）。

> 当前 `skill_runtime.gd` 被动槽仅做"已领悟校验"，**尚未读取 `weapon_mechanic` 与装备状态**——这是 G1 的核心工作。

## §8 与文档体系的衔接

- **`19-符文系统`**：本文是「骨架层」，19 是「乘数层」。19 §11.2 已交叉引用本文。
- **`21-被动技能`**：被动技能分**数值类**（武器/盾数值）与**机制类**（操作强化，如蓄力/二段跳/撞敌）。`20` 的武器命中机制与 `21` 的操作强化并列构成"被动机制"全貌，均被 B 类符文放大。
- **`15-技能与领悟`**：双轨领悟阶梯（§1.1）与属性里程碑（§3）是武器层/流派层/属性轴三者的**解锁来源**；本文 §2/§3 的流派补全需回写 15 §2。
- **`16-技术架构`**：武器定义 / 流派被动 / 机制聚合 / 分发器对应 `skill_data` / `skill_runtime` 与战斗层模块。
- **`项目实现进度评估`**：本文补全了"构筑深度"支柱的骨架明细，是评估里"已落地但需收尾"部分的细化基线。
