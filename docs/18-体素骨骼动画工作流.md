# 体素骨骼动画工作流

本文档规定体素角色模型从骨骼绑定到武器/攻击动画的完整制作流程。
所有体素骨骼模型必须同时遵守 `docs/17-体素建模工作流.md`（静态体素主体）与本文（骨骼动画）。

## 0. 与现有系统的关系

游戏角色动画系统的真实结构（详见 `character_skeleton_animation_review.md`）：

- **驱动方式**：纯 `AnimationPlayer.play(名称)` 驱动，无 `AnimationTree`。
- **武器挂载**：`BoneAttachment3D`（`bone_name="Hand.R"` 武器 / `Hand.L` 盾）→ `WeaponPlaceholder` → `EquipedItem`。
- **攻击动画选择**：`globals/combat/combat_slash_animator.gd` 的 `player_animation_name(weapon)` 按武器类型映射到专用动画名，缺失时回退到 `"slash"`。
- **武器弧线**：`CombatSlashAnimator.apply_weapon_arc()` 在挥砍期间对 `WeaponPlaceholder` 施加程序化弧线（roll/yaw/前推），叠加在骨骼动画之上。
- **轨道验收**：导出的 GLB 必须直接包含正确的轨道类型和关键帧数据；禁止在运行时修补损坏轨道。

**核心约束**：体素 `_rig.glb` 的骨骼名 + 动画名必须与 `character.glb` 对齐，才能被现有状态机与 `CombatSlashAnimator` 直接驱动。

## 1. 数据流总览

```
generate_voxel_<model_id>.py (Blender headless，固定单一模型身份)
      │  1. 按 docs/17 的独立像素尺寸表生成该模型静态 GLB
      │  2. 在该模型脚本中创建骨架（非人形必须为专属骨架）
      │  3. 在该模型脚本中显式定义 parts_by_bone 并绑定部件
      │  4. 调用 voxel_character_rig.py 的共享动作/导出机械
      │  5. 只导出该模型的 _rig.glb
      ▼
voxel_*_rig.glb  ──→  Godot (import)
      │                 │
      │                 ├── VoxelAnimationSpec (单一真相源：骨骼名 + 动画名)
      │                 └── VoxelRigValidator (运行时/测试校验)
      ▼
敌人/玩家场景：用 instance= 覆盖 character 节点指向 _rig.glb
      │
      ├── AnimationPlayer.play("slash_one_hand" / "thrust_spear" / ...)
      ├── BoneAttachment3D (Hand.R) 挂武器
      └── CombatSlashAnimator.apply_weapon_arc 程序化弧线
```

## 2. 比例尺

与 `docs/17` 完全一致：`1m = 32px`，`1px = 1/32m`。

- 骨骼坐标先用像素设计，再换算到米。人形可使用 `voxel_humanoid_rig.py` 的单位换算与骨架机械；非人形换算和骨骼定义必须由该模型的专用生成器拥有。
- 地面偏移 `GROUND_OFFSET_M = 1px`，避免穿模/Z-fighting。

## 3. 骨骼命名规范

### 3.1 人形骨架合约

17 根骨骼，命名与 `character.glb` 的人形子集对齐：

| 部位 | 骨骼名 | 武器系统用途 |
|---|---|---|
| 根 | `Root` | — |
| 髋 | `Pelvis` | — |
| 躯干 | `Torso` | 攻击躯干扭转 |
| 颈 | `Neck` | — |
| 头 | `Head` | 受击/死亡 |
| 右臂 | `UpperArm.R` `LowerArm.R` `Hand.R` | **武器挂载骨** |
| 左臂 | `UpperArm.L` `LowerArm.L` `Hand.L` | **盾挂载骨** |
| 右腿 | `UpperLeg.R` `LowerLeg.R` `Foot.R` | 跑/踢 |
| 左腿 | `UpperLeg.L` `LowerLeg.L` `Foot.L` | 跑/踢 |

> `.R` / `.L` 后缀不可更改——`BoneAttachment3D.bone_name` 与导出动画的轨道路径均依赖此命名。

完整定义见 `tools/voxel_humanoid_rig.py` 的 `REFERENCE_BONES`。

### 3.2 非人形

非人形不存在可共享的生物骨架表。每个通过独立重建验收的模型都在自己的生成器中定义专属骨架：

| 模型 | 骨骼数 | 关键骨 |
|---|---|---|
| dragon | 14 | `Torso` `Neck1` `Neck2` `Head` `Tail1-3` `Wing.L/R` `FrontLeg.L/R` `BackLeg.L/R` |

表中名称是 dragon 的动画合约，不是其他生物可复用的骨架模板。新的非人形模型必须在自己的 `generate_voxel_<model_id>.py` 中独立定义骨骼位置、层级和 `parts_by_bone`。

## 4. 动画名规范（单一真相源）

`globals/visual/voxel_animation_spec.gd` 是动画名 + 骨骼名的**唯一真相源**。Blender 脚本按此规格制作，Godot 校验器按此规格检查。

### 4.1 基础动作动画（人形 + 非人形通用）

```
idle  run  hurt  stunned  death  kick  lift  pickup  throw_weapon  throw_furniture  block
```

### 4.2 武器攻击动画（人形必需）

`CombatSlashAnimator.player_animation_name(weapon)` 的映射目标。缺失时回退到 `"slash"`，导致对应武器攻击外观与通用挥砍相同。

| 动画名 | 武器类型 | 帧数 | 说明 |
|---|---|---|---|
| `slash` | 通用兜底 | 10 | 对角挥砍（所有武器的基础回退） |
| `slash_one_hand` | 单手剑/斧/钉锤 | 10 | 右手对角挥砍 |
| `slash_heavy` | 双手大剑/战锤 | 14 | 双手过头劈砍 |
| `slash_dagger` | 匕首 | 8 | 快速双手交替连斩 |
| `thrust_spear` | 长矛 | 10 | 向前突刺 |
| `bash_shield` | 盾 | 8 | 左手盾击前推 |
| `claw_swipe` | 徒手（无武器） | 8 | 双手爪击 |

### 4.3 姿态动画

| 动画名 | 帧数 | 说明 |
|---|---|---|
| `default` | 1 | 零姿态（所有骨骼归零），校验基准 |
| `hold_weapon` | 1 | 装备武器时的待机持握姿态（右臂前抬握持，左臂备盾） |

### 4.4 人形完整必需集

基础(11) + 武器攻击(7) + 姿态(2) = **20 个动画**。

非人形最小集：基础(11) + `slash` + `claw_swipe` + `default` = **14 个动画**。

## 5. 攻击动画设计原则

所有武器攻击动画遵循**蓄力 → 挥击 → 收招**三段结构，与现有 `slash` 一致：

```
帧1(起势) → 帧N(蓄力峰值) → 帧M(挥击命中) → 末帧(回到 rest)
```

- **起止归零**：第 1 帧与末帧均为 rest（`{}`），保证与 `idle` 之间的 blend 平滑。
- **命中段对齐**：`CombatSlashAnimator` 用 `PLAYER_HIT_START=0.28` / `PLAYER_HIT_END=0.78` 归一化进度判定命中窗口。动画的"挥击"关键帧应落在 28%~78% 区间。
- **程序化弧线叠加**：`apply_weapon_arc` 会对 `WeaponPlaceholder` 施加额外 roll/yaw/前推。骨骼动画负责身体/手臂的大动作，弧线负责武器的精细摆动——两者**叠加**而非冲突。
- **旋转单位**：Blender 脚本中 `make_action` 的 `rot` 参数为**度数**，由 `_rad()` 转 弧度。

## 6. 持武器姿态（hold_weapon）

`hold_weapon` 是装备武器后的待机姿态，解决"武器贴在腿侧"的违和感：

```
UpperArm.R  rot(-30, 0, 25)   右臂前抬外展
LowerArm.R  rot(-40, 0, 0)    前臂内屈握持
Hand.R      rot(0, -20, 20)   腕角对齐握柄
UpperArm.L  rot(-20, 0, -30)  左臂前抬备盾
LowerArm.L  rot(-50, 0, 0)    前臂上屈
```

> 当前 `player_state_moving.gd` / `enemy_state_moving.gd` 在持武器时仍播 `idle`。未来可在装备武器后切换为 `hold_weapon` 作为待机，无需改动画数据。

## 7. 制作流程（Blender 管线）

### 7.1 人形生物

```bash
# 只运行 goblin 的独立单模型入口
D:/123/blender/blender.exe --background --python tools/generate_voxel_goblin.py
```

专用生成器不得导入一份旧 GLB 再套用已删除的通用 rig 改造流程。现行流程为：
1. 在该模型脚本中生成静态主体，并保留该模型的部件对象列表。
2. 创建符合该模型比例的骨架；共享人形骨架 API 只提供骨骼命名与机械操作，不提供身体或 silhouette 模板。
3. 在该模型脚本中显式声明 `parts_by_bone`，不得通过共享模型注册表推断绑定。
4. 调用 `voxel_character_rig.py` 的 `build_all_actions()` / `build_weapon_actions()` 共享动作机械。
5. 校验骨骼、绑定、轨道类型和关键帧数据后，只导出该模型的 `_rig.glb`。

禁止使用 `all`、`humanoid`、档位、通配符、目录、逗号分隔 ID，或用循环连续调用多个生成器。

### 7.2 非人形生物

```bash
D:/123/blender/blender.exe --background --python tools/generate_voxel_dragon.py
```

该模型的专用生成器必须自行定义静态主体、专属骨骼、骨骼层级和显式 `parts_by_bone`。
`voxel_character_rig.py` 只能提供共享动作数据写入与导出机械，不得持有生物注册表、通用生物骨架或绑定表。

### 7.3 添加新动画

1. 在 `globals/visual/voxel_animation_spec.gd` 的对应数组中**先登记动画名**。
2. 可复用的动作写入机械在 `tools/voxel_character_rig.py` 的 `build_weapon_actions` /
   `build_all_actions` 中维护。非人形骨架、绑定和模型专属动作必须在该模型自己的生成器中接入，不得建立模型注册表。
3. 只重新运行该模型的 `tools/generate_voxel_<model_id>.py` 生成 static + `_rig.glb`。
4. 运行第 8 节的校验测试。

> **铁律**：动画名必须先在 `VoxelAnimationSpec` 登记，否则校验测试无法发现缺失。规格 → 数据 → 校验，顺序不可颠倒。

## 8. 校验与测试

### 8.1 自动校验（gdUnit4）

```bash
# 校验所有体素 _rig.glb 的骨骼名 + 动画名是否齐全
"D:/123/Godot_v4.7-stable_mono_win64.exe" --headless -s tests/gdunit4_runner.gd -- --ignoreHeadlessMode -a tests/gdunit/voxel_rig_animation_test.gd
```

测试内容（`tests/gdunit/voxel_rig_animation_test.gd`）：
- `VoxelAnimationSpec` 规格本身的完整性（数组非空、无重复、武器挂载骨存在）。
- 每个人形 `_rig.glb`：含全部 17 必需骨骼 + 20 必需动画。
- 每个非人形 `_rig.glb`：含全部 14 必需动画。
- `VoxelRigValidator` 对缺失项的正确报告。

### 8.2 轨道拒收标准

`kick` / `run` 等旋转动画的腿部轨道必须在导出 GLB 中就是 `ROTATION_3D`。任何被存为 `SCALE_3D`、`BLEND_SHAPE` 或含 NaN/非归一化四元数的轨道都必须阻止验收。修复必须发生在该模型的生成器或 `voxel_character_rig.py` 的共享动作/导出机械中，然后仅重新导出和验证该模型。不得依赖运行时修补。

### 8.3 三视图验证（可选）

武器攻击动画的三视图截取（确认挥砍轨迹合理）可复用 `tools/voxel_prop_three_view_capture.gd` 的思路扩展，当前非强制。

## 9. 接入场景（未来工作）

当前所有敌人场景仍用 `character.glb`。接入体素模型的步骤（阶段 B，需逐个验证）：

1. 在敌人场景（如 `goblin.tscn`）中将 `character` 节点的 `instance=` 指向 `voxel_*_rig.glb`。
2. 确认 `BoneAttachment3D` 的 `bone_name` 与体素骨骼名一致（`Hand.R` / `Hand.L`）。
3. `PhysicalBoneSimulator3D` 的 bone_offset / 质量按体素比例重新调整。
4. 运行 `voxel_rig_animation_test.gd` + 场景冒烟测试确认无回归。

> 非人形模型是否能挂载武器取决于该模型自己的已验收骨架合约。例如 dragon 无 `Hand.R`，使用 `claw_swipe` 徒手攻击；不得从其他非人形模型推断其骨骼或绑定。

## 11. 第一人称 ViewModel 动画

第一人称武器是本地视觉层，和第三人称骨骼动画使用不同资源：

- `view_model.tscn` 固定为 `BobPivot → AimPivot → ActionPivot → WeaponSocket`；只有 AnimationPlayer 写入 `ActionPivot`。
- `view_model_animation_library.tres` 保存 `vm_*` 时间轴。近战/蓄力由状态机提供归一化进度采样；投射物和命中窗口仍由 PlayerState 与 CombatSlashAnimator 决定。
- 使用 `scenes/debug/view_model_animation_preview.tscn` 可从 WeaponRegistry 选真实武器并预览动作；该场景不写入 weapons.json，也不导出 GLB。
- 不要把 ViewModel 动画轨道指向摄像机、AimPivot、BobPivot 或运行时 WeaponModel。武器安装修正只放在 WeaponSocket。

| 文件 | 职责 |
|---|---|
| `globals/visual/voxel_animation_spec.gd` | 动画名 + 骨骼名单一真相源 |
| `globals/visual/voxel_rig_validator.gd` | GLB 运行时/测试校验器 |
| `globals/combat/combat_slash_animator.gd` | 攻击动画选择 + 程序化弧线 |
| `tools/generate_voxel_<model_id>.py` | 固定身份的单模型 Blender 入口；只写该模型 static + rig + 预览 |
| `tools/voxel_character_rig.py` | 单模型生成器调用的共享动作写入与导出机械 |
| `tools/voxel_humanoid_rig.py` | 人形骨骼合约与机械绑定助手；`parts_by_bone` 仍由每个模型脚本显式定义 |
| `tests/gdunit/voxel_rig_animation_test.gd` | 规格与 GLB 校验测试 |
| `character_skeleton_animation_review.md` | 现状审查报告（背景） |
