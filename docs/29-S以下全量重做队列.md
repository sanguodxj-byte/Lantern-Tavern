# A→D 单模型重建队列

> 政策修正（2026-07-21）：**档位与验收状态分开记录；A/B/C/D 必须按 A→B→C→D 的顺序逐只独立重建、测试和视觉验收，完成后才可进入 `ACCEPTED_IDS`，未验收模型继续隔离。**
> 早先以批量脚本得到的 `ok=37 failed=0` 只证明当时脚本运行结束，不能证明每只模型独立设计或达到质量门槛，现不再作为完成证据。
> `skeleton` 与 `troll` 已从 S 档下调到 A 档；独立验收不会改变其历史档位。

## 单模型纪律

1. 当前只允许一个活动模型 ID；该模型完成全部 DoD 后才能开始下一只。
2. 每只模型必须由 `tools/generate_voxel_<model_id>.py` 独立定义尺寸、silhouette、语义部件、材质和固定输出身份。
3. 禁止 `all`、档位选择器、通配符、目录扫描/循环、逗号分隔 ID、多 ID 参数，及在一个命令、脚本、编辑器工具或 CI job 中连续调用多个生成器。
4. 共享模块只能提供体素原语、校验、渲染和 rig 机制；不得保存模型注册表或批量输出循环。
5. A 全部完成前不得开始 B；B 全部完成前不得开始 C；C 全部完成前不得开始 D。

## 每只模型的完成定义（DoD）

- [ ] 独立像素尺寸表与专属 silhouette，不复用模板换皮。
- [ ] 专用生成器拒绝其它模型 ID，只写自身 static GLB、rig GLB 和自身预览。
- [ ] `assert_parts_voxel_assembly_valid` 通过：无正体积重叠且所有静态部件面接触连通。
- [ ] 该模型 companion gdUnit 测试及角色 overlap/rig 校验通过。
- [ ] 使用精确 `--asset=<model_id>` 新生成 front/side/top 结构图，并确认文件新鲜、可读、非空。
- [ ] 检查独立真 3D front/side/top/preview 的材质、光照、遮挡和整体体积。
- [ ] 人工确认无变小、漂浮、角接触、z-fighting、错位效果原点或劣质旧模型覆盖。

```bash
"D:/123/Godot_v4.7-stable_mono_win64.exe" --headless --path "D:/123/Lantern Tavern" --script res://tools/voxel_prop_three_view_capture.gd -- --asset=skeleton
```
