## 地牢生成「游戏性」优化（落地 + 测试全绿）
用户确认"合理"指游戏性而非代码后，实施了 3 项优化（此前已在 2026-07-17.md 记录设计）：
1. **掉落多样性**：`dungeon_spawn_planner.gd` 新增 `ZONE_MATERIAL_POOLS`（每区域 3 候选材料），`plan_item_spawns` 改用 `_seeded_rng(layout, 0x4D4154)` + `_pick_material_from_pool` → 同 seed 同掉落、跨房有变化、跨 seed 不同。替代旧"每房恒 blackberry"。
2. **敌人数方差 + 深度梯度**：`_calc_room_enemy_count(zone_cfg, floor_cell_count, depth, rng)` 改为 base±1 随机 + `int(depth/12)` 梯度（越深越多）；用 `compute_floor_distance_field` + `depth_of_room_with_field` 求房间深度。打破"每房恒 2 敌"。
3. **喘息房**：`dungeon_hazard_planner.gd` 新增 `BREATHER_ROOM_COUNT=2`/`MIN_ROOMS_FOR_BREATHER=6` + `_select_breather_rooms`（BFS 距离取最靠近出生点的普通房设为无陷阱）；plan() 对喘息房 `continue` 跳过陷阱。房间 <6 时不设喘息房避免掏空陷阱。
